# QA Test Plan — gmlegends.kz

**URL**: https://gmlegends.kz  
**Окружение**: AlmaLinux 9 VPS (4 CPU / 4 GB RAM), Docker  
**Браузеры**: Chrome (desktop + mobile), Safari (iOS), Firefox

---

## 1. Регистрация и Авторизация

### 1.1 Регистрация

| # | Шаг | Ожидаемый результат |
|---|------|---------------------|
| 1 | Открыть https://gmlegends.kz | Страница логина загружается, логотип GML отображается, замок HTTPS зелёный |
| 2 | Нажать "Создать новый профиль" | Открывается форма регистрации |
| 3 | Оставить все поля пустыми → "Зарегистрироваться" | Ошибка валидации, кнопка не отправляет |
| 4 | Ввести никнейм `testqa1`, телефон `+77001234567`, пароль `123` | Ошибка: пароль слишком слабый (мин. 6 символов, цифра + заглавная) |
| 5 | Пароль `Test1234` | Регистрация успешна, редирект на Dashboard |
| 6 | Повторить с тем же телефоном | Ошибка: пользователь уже существует |
| 7 | Зарегистрировать `testqa2` с email вместо телефона | Проверить что email тоже работает |

### 1.2 Логин / Логаут

| # | Шаг | Ожидаемый результат |
|---|------|---------------------|
| 1 | Ввести неверный пароль → "Играть!" | Ошибка авторизации |

| # | Шаг | Ожидаемый результат |

## 2. Дашборд и Дорожная карта

### 2.1 Начальный тест

| # | Шаг | Ожидаемый результат |
|---|------|---------------------|
| 1 | Зайти на Dashboard новым юзером | Предлагается пройти начальный тест |
| 2 | Пройти тест, отвечая на все вопросы | Результат отображается, уровень определён |

### 2.2 Дорожная карта — уроки

| # | Шаг | Ожидаемый результат |
|---|------|---------------------|
| 1 | Проверить что Неделя 1 доступна, Недели 2-12 заблокированы (серые/замок) | Блокировка работает |

| 4 | Пролистать урок до конца | Появляется мини-тест по уроку |
| 5 | Ответить правильно на тест → "Завершить" | Урок отмечен галочкой, прогресс-бар обновляется |
| 6 | Завершить все уроки Недели 1 | Неделя 2 разблокируется |

## 3. Игра с ботами

### 3.1 Выбор сложности

| # | Шаг | Ожидаемый результат |
|---|------|---------------------|
| 1 | Перейти в "Играть с ботом" | Страница выбора ботов загружается |
| 2 | Выбрать "Новичок", контроль 5 мин | Партия начинается, доска отображается |
| 3 | Сделать ход e2-e4 (drag & drop) | Бот отвечает за < 1 секунду (depth 2) |
| 4 | Сыграть 5-10 ходов | Бот играет слабо, делает нелогичные ходы |
| 5 | Начать новую партию → "Мастер" | Бот отвечает за 1-2 секунды, играет заметно сильнее |
| 6 | Начать новую партию → "Легенда" | Бот думает 1-3 секунды, играет на уровне ~2600 Elo |
| 7 | Нажать "Сдаться" | Партия завершается, результат записывается |

### 3.2 Разница в силе

| # | Проверка | Ожидаемый результат |
|---|----------|---------------------|
| 1 | Новичок: поставить мат в 2 хода (Scholar's mate) | Бот НЕ защищается (слабая игра) |
| 2 | Легенда: поставить мат в 2 хода | Бот ДОЛЖЕН защититься |
| 3 | Сравнить ходы ботов из одной позиции | Разные уровни → разные ходы |

## 4. Анализ партий

### 4.1 Анализ из истории

| # | Шаг | Ожидаемый результат |
|---|------|---------------------|
| 1 | Перейти на страницу "Анализ" | Список сыгранных партий (из п.3) отображается |
| 2 | Нажать на любую партию | Загружается анализ: доска, ходы, оценки |
| 3 | Перемотать ходы стрелками ← → | Доска обновляется, evaluation bar меняется |
| 4 | Проверить список ошибок (blunders) | Ходы-ошибки выделены красным, лучшие ходы показаны |
| 5 | Нажать на ошибку в списке | Доска перематывается к позиции ошибки |

### 4.2 PGN текст (Сканер позиции)

| # | Шаг | Ожидаемый результат |
|---|------|---------------------|
| 1 | Перейти на "Сканер позиции" | Два поля: текстовое PGN и загрузка фото |
| 2 | Вставить PGN: `1. e4 e5 2. Nf3 Nc6 3. Bb5 a6 4. Ba4 Nf6 5. O-O` | Текст принимается в textarea |
| 3 | Нажать "Анализировать PGN" | Анализ загружается: доска, ходы с оценками, ошибки, рекомендации |
| 4 | Использовать навигацию (prev/next, slider, клавиши) | Позиция на доске меняется корректно |
| 5 | Вставить невалидный PGN: `1. Zz4 abc` → "Анализировать" | Ошибка с понятным сообщением |
| 6 | Вставить пустой текст → "Анализировать" | Ошибка валидации |

### 4.3 PGN фото

| # | Шаг | Ожидаемый результат |
|---|------|---------------------|
| 1 | Загрузить фото шахматного бланка (чёткое) | Фото принимается, имя файла отображается |
| 2 | Нажать "Распознать фото PGN" | Происходит распознавание → PGN текст → анализ |
| 3 | Загрузить фото НЕ бланка (котик) | Ошибка распознавания или пустой результат |
| 4 | Загрузить файл > 16 МБ | Ошибка: файл слишком большой |

## 9. HTTPS и безопасность

| # | Проверка | Ожидаемый результат |
|---|----------|---------------------|
| 1 | Открыть http://gmlegends.kz | Автоматический 301 редирект на https:// |
| 2 | Проверить сертификат (кликнуть на замок) | Let's Encrypt, валидный, gmlegends.kz + www |
| 3 | Открыть DevTools → Network → любой API запрос | Все запросы идут на https://gmlegends.kz/api/ |
| 4 | Проверить заголовки ответа | `Strict-Transport-Security`, `X-Frame-Options: SAMEORIGIN`, `X-Content-Type-Options: nosniff` |
| 5 | Открыть https://www.ssllabs.com/ssltest/ → ввести gmlegends.kz | Рейтинг A или A+ |
